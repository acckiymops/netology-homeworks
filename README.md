## 1. Создана ВМ в эконом конфигурации
Intel Ice Lake. 20% vCPU — прерываемые ВМ
Публичный IP-адрес
Intel Ice Lake. RAM — прерываемые ВМ
Стандартное сетевое хранилище (HDD)


## 2. 
* Высоконагруженная база данных MySql, критичная к отказу - физические сервера.
Доступ к сетевой карте, низкие задержки (критично для репликации), Высокая производительность дискового i/o, все ресурсы выделены под БД

* Различные web-приложения - виртуализация уровня ОС. Быстрота развертывания, изоляция, эфективность использования ресурсов.
* Windows-системы для использования бухгалтерским отделом - паравиртуализация.
* Системы, выполняющие высокопроизводительные расчёты на GPU - физические сервера. Прямой доступ к GPU для высокой производительности

## 3.
* vSphere. Совместимость с Windows-инфраструктурой, наличие инструментов для репликаций и бекапа
* KVM. Бесплатное нативное решение
* Hyper-V. Максимальная совместимость с Windows
* VirtualBox. Простота в создании и удалении ВМ, кроссплатформеность

## 4.
* различная логика развертывания в разных системах
* увеличение нагрузки на эксплуатацию и администраторов систем (необходимо знать несколько систем управления, а не одну)
* увеличение затрат бизнеса на техподдержку систем
* сложность мониторинга
* проблемы с миграцией ВМ между системами

  Для минимизации проблем необходимо:
* внедрить единую систему мониторинга
* централизованный сбор логов
* использование инструментов ansible и terraform для развертывания и управления инфраструктурой по принципу IaC

  Создание гетерогенной среды крайне нежелательно
